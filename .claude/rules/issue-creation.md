# Issue作成

他のコーディングエージェントが不明なことなく作業できるように、Issueは完全明確に記述する。

## 必須セクション

| セクション | 内容 |
|-----------|------|
| 概要 | 問題または機能の簡潔な説明（1-2文） |
| 再現手順 | 番号付きステップで詳細に記述。期待動作と実際の動作を明記 |
| 環境 | OS、ツールのバージョン、関連ブランチ/コミット |
| 完了条件 | チェックリスト形式で検証可能な条件を列挙 |

## 推奨セクション（該当する場合）

| セクション | 内容 |
|-----------|------|
| 発生タイミング | いつから発生したか（特定のコミット/PR以降など） |
| 原因の推測 | 考えられる原因を技術的に列挙 |
| 調査方法 | デバッグ手順、確認すべきログ |
| 関連ファイル | 変更が必要そうなファイルパス |
| 技術的背景 | 関連するコードスニペット、アーキテクチャの説明 |

## 完全明確の基準

Issueは以下の質問に答えられる状態であること：

1. **何が問題/要件か？** → 概要で明確
2. **どうやって再現/確認するか？** → 再現手順で明確
3. **どこを見るべきか？** → 関連ファイルで明確
4. **何をもって完了とするか？** → 完了条件で明確
5. **なぜ起きているか？** → 原因の推測で方向性を示す

## コード例の記載

技術的背景には、関連するコードスニペットを含める：

```markdown
### 現在の実装（ファイル名:行番号付近）

\`\`\`swift
// 問題のあるコード or 変更が必要なコード
\`\`\`
```

## 悪い例と良い例

### 悪い例

```markdown
## 概要
ポップアップが動かない

## 完了条件
- [ ] 動くようにする
```

問題点：
- 再現手順がない
- 環境情報がない
- 完了条件が曖昧

### 良い例

```markdown
## 概要
Xcodeの+ボタンでポップアップを表示後、ヒントモードを起動するとフリーズする。

## 再現手順
1. Portalを起動
2. Xcodeを開く
3. Fキーでヒントモード起動
4. +ボタンのヒントを選択
5. ポップアップ表示後、再度Fキーを押す
6. **期待**: ポップアップ内にヒントラベル表示
7. **実際**: フリーズする

## 環境
- macOS 15.0+
- Xcode 16.2+
- ブランチ: feature/#132（コミット abc123 以降）

## 関連ファイル
- `Portal/Crawlers/NativeAppCrawler.swift` - 要素走査
- `Portal/HintMode/HintModeController.swift` - モード制御

## 原因の推測
1. クローラーの無限ループ
2. ポップアップウィンドウの検出漏れ

## 完了条件
- [ ] ポップアップ内の要素にヒントラベルが表示される
- [ ] フリーズしない
- [ ] 既存動作に影響がない
```
